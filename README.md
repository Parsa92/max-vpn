# MAX VPN - Telegram VPN Reselling Platform

A fully automated Telegram VPN Reselling platform that purchases servers from a source bot, rebrands subscription configs, and delivers them to customers via HTTPS.

## Features

- **Automated Server Purchase**: Buys VPN servers from @MMDLeecherNimBot using Pyrogram userbot
- **Config Rebranding**: Replaces "MMDLeecher" with "max_v2connect" in subscription configs
- **HTTPS Delivery**: FastAPI serves rebranded configs at `https://your-domain/sub/{order_id}`
- **Manual Payment Approval**: Admin approves wallet recharges via inline buttons
- **Admin Panel**: Manage users, orders, prices, and trigger link refresh
- **Daily Link Refresh**: Automatically updates subscription links for all active orders
- **AI Fallback**: Gemini AI fallback when main automation fails
- **Refund on Failure**: Automatic refund if server purchase fails

## Tech Stack

- **Bot Framework**: aiogram 3.x (sales bot)
- **MTProto**: Pyrogram 2.x (userbot automation)
- **Database**: PostgreSQL + asyncpg
- **Queue**: Redis + arq
- **API**: FastAPI + Uvicorn
- **Proxy**: Nginx (SSL termination)
- **AI**: Google Gemini (fallback automation)
- **Orchestration**: Docker Compose

## Quick Start

### 1. Clone & Install

```bash
git clone https://github.com/parsa-salar/max-vpn.git
cd max-vpn
sudo bash install.sh
```

The installer will:
- Install Docker & Docker Compose (if not present)
- Create `config.json` from template
- Generate Pyrogram session string
- Build and start all services

### 2. Configure

Edit `config.json` with your details:

```json
{
  "telegram": {
    "bot_token": "YOUR_BOT_TOKEN",
    "api_id": "YOUR_API_ID",
    "api_hash": "YOUR_API_HASH",
    "admin_id": "YOUR_TELEGRAM_USER_ID"
  },
  "payment": {
    "bank_card_number": "YOUR_BANK_CARD"
  },
  "gemini": {
    "api_key": "YOUR_GEMINI_API_KEY",
    "model": "gemini-2.5-flash"
  },
  "server": {
    "host": "YOUR_SERVER_IP_OR_DOMAIN",
    "use_valid_ssl": false
  }
}
```

### 3. Get Your Credentials

| Credential | Where to get it |
|------------|-----------------|
| `bot_token` | Message @BotFather on Telegram → `/newbot` |
| `api_id` | Go to [my.telegram.org](https://my.telegram.org) → API development tools |
| `api_hash` | Same page as api_id |
| `admin_id` | Message @userinfobot on Telegram to get your user ID |
| `gemini_api_key` | Go to [aistudio.google.com](https://aistudio.google.com) → Get API key |
| `pyrogram_session_string` | Auto-generated by installer, or run `python3 generate_session.py` |

### 4. Start

```bash
docker compose up -d
```

## Bot Commands

| Command | Description |
|---------|-------------|
| `/start` | Welcome message & main menu |
| `/admin` | Admin panel (admin only) |

## User Flow

1. User sends `/start` → sees main menu
2. Clicks "🛒 خرید سرور" → selects a plan (10/25/50/100 GB)
3. Balance is deducted → order is queued
4. Worker buys server from source bot
5. Configs are rebranded and delivered via HTTPS link
6. User receives subscription link

## Admin Panel

Send `/admin` to access:
- 👥 **Users**: List all registered users
- 📋 **Pending Orders**: View orders being processed
- ✅ **Completed Orders**: View successful orders
- 💰 **Change Prices**: Update plan prices (persists to config.json)
- 🔄 **Refresh Links**: Trigger manual link refresh for all orders

## Payment Flow

1. User clicks "💰 کیف پول" → "💳 شارژ کیف پول"
2. Sends payment to bank card number
3. Uploads receipt photo
4. Admin receives receipt with approve/reject buttons
5. Admin enters custom amount → balance is updated

## SSL Setup

- **IP Address** (`use_valid_ssl: false`): Self-signed certificate (browser warning)
- **Domain** (`use_valid_ssl: true`): Let's Encrypt certificate (auto-renewed)

## Troubleshooting

### View Logs
```bash
docker compose logs -f                  # All services
docker compose logs -f userbot-worker   # Worker only
docker compose logs -f sales-bot        # Bot only
```

### Restart Services
```bash
docker compose restart
```

### Rebuild After Code Changes
```bash
docker compose down
DOCKER_BUILDKIT=0 docker build --network=host -t max-vpn-api -f Dockerfile.api .
DOCKER_BUILDKIT=0 docker build --network=host -t max-vpn-bot -f Dockerfile.bot .
DOCKER_BUILDKIT=0 docker build --network=host -t max-vpn-worker -f Dockerfile.worker .
docker compose up -d
```

### Common Issues

| Issue | Solution |
|-------|----------|
| Bot not responding | Check `docker compose logs sales-bot` |
| Orders failing | Check `docker compose logs userbot-worker` |
| API not accessible | Check `docker compose logs api` |
| Nginx restarting | Normal if no SSL cert yet (IP mode) |
| Database connection error | Check `docker compose logs postgres` |
